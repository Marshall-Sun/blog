你好是孙健强同学是吧？

对。

你是现在在哪儿呢？

东北大学，沈阳。

那你打算是什么时候实习或者说实习多长时间？

理论上期末考之后就可以去实习，可以实习 6 个月或者以上。

行好。你现在是本科是吧，大四还是大三？

大三了

大三是吧，行，那我就根据你的简历问几个问题。我看你做前端主要是 React 和 Angular 是吧？

对

那我问一个 React 的问题，你写 React 的时候有没有做一下状态管理的一些东西啊，比如说 Redux 之类的

没有用到

就是用 React 简单写一下页面是吧

对，主要是利用了 ant design 的一个数据可视化。React 本身其实相对比较少用到，主要是写 JavaScript。

有没有涉及到数据通信，就是客户端和服务器的通信？咱们用什么方式实现的？

那个就是接入了 Firebase 的实时数据库，然后 Firebase 是有相关的接口，就是直接调用。一开始用的是 Promise，然后我用 async 函数改进了一下，把代码搞得简洁了一点。

async 什么作用呢？

async 函数其实就是生成器函数的语法糖，它要和 await 结合，使用 await 语句其实就是 then 的语法糖，大概就是这些。它其实就是异步编程的一个方法

我用 Promise 也可以异步编程，我为什么要用 async？它相对于 Promise 有什么优点？

如果用 Promise 就会要使用 then、catch、finally。如果使用 Promise 的话，我组织一下语言。

使用 Promise 的话，其实我觉得好像应该是会代码冗余，不管什么操作一眼看上去都会有很多 then，就原来的语义变得不清楚。如果用 async 函数的话，它可以用同步的语义编写异步的代码，所以就相对更好一点。

开发的时候有没有遇到跨域问题？

有。在我们的项目中主要是用 Proxy 的方式来解决跨域。

因为跨域主要是浏览器跟服务器出现的问题，如果服务器跟服务器请求就不存在跨域的问题。

Angular 提供了这个方法，只要编写 proxy-config，它就会自动使用正则表达式来根据 HTTP 请求的 URL 来自动帮你转发，这样就方便很多。叫 proxy 是吧？对。

咱们这个项目开发完了部署到哪里去呢？

我有几个项目，Angular 的话现在在最终测试，所以现在还是部署在我们学院的服务器上。

React 项目的话是交付给北卡电力公司使用的，应该是部署在他们的服务器上，具体我也不是很清楚。

我的博客是部署在 Github Pages 上的，它主要是因为它是免费的，还有个人课表也是 Github Pages。

咱们在开发的时候有没有用过 Git 来管理项目？都常用什么 Git 命令？

Git commit 就是将暂存区的内容提交到了本地仓库
Git add 就是把工作区修改的内容提交到了暂存区。
如果要跟 Github 同步的话，主要是有两个，一个是 Git push 和 Git pull。
Git push 是将本地仓库推送，然后 Git pull 是从远程拉取。行。

有没有了解过 Webpack 一些配置？

我知道我们 Angular 的项目中用到了，但是具体不是我来处理的。然后我本身的项目那两个都是 Github Pages，也相对比较少，用的都是直接用 cli 部署上去，就也没有说什么详细的配置，没有很深的了解。

做 Angular 的时候，我看有写到有异步路由守卫，这个是解决什么问题？

异步路由守卫这个就是他当用户访问某一个 URL 的时候，我们要判断 URL 是否应该被当前用户访问，因为不同的用户有不同的权限，比如说管理员，用户还有教师，学生肯定是不能访问课程管理界面，这个我们就要判断他是否可以访问，如果不能访问就要提醒他跳转到首页。

异步路由守卫的话主要是首先要判断当前用户的身份，然后把这个身份发送给后端，让后端来判断它是否用了权限，然后从后端拿到 JSON 的相关的那些布尔值，然后把这个东西给 Angular，然后 Angular 就会判断。行。

你一般会常用什么样的页面布局方式？

页面布局现在主要是用 flex

如果要实现水平垂直居中的话，怎么来实现？

首先要把父元素设置为 display: flex，然后 justify-content: center，然后 align-items: center

就是主轴居中，垂直轴的居中

响应式的话，是用什么来实现的响应式布局？

我们主要是 angular 的响应式布局，主要是用媒体查询和三个相对长度单位来实现。

媒体查询就是针对不同的媒体类型进行不同的样式，它设置几个节点，在某个节点区间之间就会运用对应的 CSS 样式，这样就可以同一套 CSS 代码，适应不同屏幕的尺寸。

除了媒体查询，要结合三个相对长度单位就是百分比和 iem 和 视口，这样就可以方便的实现响应式布局。好的。

Express 我看你简历上有写，这个使用到了吗？

对，主要是对 MySQL 的读写。在我们学校有一些前后端交互的项目，主要是我用这个来做后端。

主要用来读写 MySQL 是吧？对。

AntV 这个框架是用来做什么的？

AntV 就是 Ant Design 的一部分，它是主要用于数据可视化，我们主要用它来将数据绘制成折线图，它里面提供了一个折线图的方法，我们就调用了它，然后绘制了折线图。

可视化相关的是吧。对。就类似于 Echarts。

React 的 hook 有使用过吗？

有，我想一下，最近比较少写 react。

componentDidMount，好像只使用到了这一个，就是用来初始化界面，就是异步

咱们做页面有没有涉及到表单的一些操作？

有，主要是用了 Angular 的响应式表单。

当时技术选型的时候，这个前端框架是怎么选择 Angular 的？

当时我进实验室的时候，实验室主要有七八个研究生，我们导师就想用一个大家最熟悉的框架，angular 当时是比较多人会用的，而且 Angular 当时是 Google 在推的，我们就自然而然的使用了 Angular。

React 和 Vue 当时我都不是很了解，所以我也只是先学了 Angular，就用了 Angular。

父子组件之间通信的话，怎么一种方式？

是哪个框架呢？比如说 react 的话

react 的话主要是用 Ref，如果是父组件要调用子组件的方法的话，首先要用 Ref

首先创建 Ref 之后，那里在子组件的 JSX 那里使用，然后父组件就可以通过 Ref.current 调用子组件的方法

如果父组件将参数直接传到子组件的话，就直接在 JSX 那里传入就可以了。

刚才说到就是 Express 来读写 MySQL，怎么传给前端呢？

express 可以就是监听一个端口之后，通过 URL 来展示一个 get 请求或者 POST 请求，然后前端就通过发起 HTTP 请求就可以。

就是通过一个具体的 get 链接或者 post 链接又发起 HTTP 请求。

你前端怎么发起 HTTP 请求？

Angular 的话他提供了 HttpModule 这个模块，就直接从 HTTP 调用 get 方法就可以直接发起 HTTP 请求，然后在它的 subscribe 回调函数里就可以这样用，或者是我经常常用的方法，在 get 之后调用它的 toPromise 方法，然后就可以用 Promised.then 或者 async 函数用 await。这样就可以之前说的，用同步语义来编写异步法。

React 的话大同小异，它主要是用的 fetch 方法
