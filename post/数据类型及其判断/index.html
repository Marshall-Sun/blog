数据类型及其判断
发表于 2020-11-14|前端知识JavaScript
字数总计:1.2k|阅读时长: 4 分钟

数据类型种类

八种，String, Number, Boolean, Object, Undefined, Null, Symbol, BigInt。

Symbol 代表创建后独一无二且不可变的数据类型，它主要是为了解决可能的全局变量冲突。

BigInt 是一种数字类型数据，它可以表示任意精度的整数，故可以安全地存储和操作超出 Number 范围 [-(2^53 - 1), 2^53 - 1] 的整数。

Undefined 与 Null 的具体区别可以参考 关于 Undefined 与 Null。

数据类型存储

在数据结构中，栈中数据的存取方式为先进后出。而堆是一个优先队列，按优先级来进行排序。完全二叉树是堆的一种实现方式。

在操作系统中，内存被分为栈区内存和堆区内存。

  • 栈区内存由编译器自动分配释放，存放函数的参数值，局部变量的值等。其操作方式类似于数据结构中的栈。

  • 堆区内存一般由程序员分配释放，若程序员不释放，程序结束时可能由垃圾回收机制回收。

栈储存着原始数据类型，共七种：String, Number, Boolean, Undefined, Null, Symbol, BigInt。存储占据空间小、大小固定，属于被频繁使用数据，所以放入栈中存储。

堆储存着引用数据类型，共一种：Object。存储占据空间大、大小不固定。如果存储在栈中，将会影响程序运行的性能；引用数据类型在栈中存储了指针，该指针指向堆中该实体的起始地址。当解释器寻找引用值时，会首先检索其在栈中的地址，取得地址后从堆中获得实体。

关于 Number 的存储

Number 是基于 IEEE 二进制浮点数算术标准（IEEE 754），使用 64 位固定长度来表示，也就是标准的 double 双精度浮点数。其中的 1 位表示符号位，11 位用来表示指数位，剩下的 52 位用来表示尾数位。

这样存储优点是可以统一处理整数和小数，节省存储空间。

为什么 0.1 + 0.2 !== 0.3

由于只能存储 52 位尾数，0.1 存到内存中再取出来转换成十进制时，就不是原来的 0.1，会出现精度缺失。

0.1 和 0.2 都转化成二进制后再进行运算
0.1 --> 0.0001100110011001100110011001100110011001100110011001101 +
0.2 --> 0.001100110011001100110011001100110011001100110011001101  =
        0.010011001100110011001100110011001100110011001100110011

转成十进制正好是 0.30000000000000004，不等于 0.3

为什么 0.2 + 0.3 === 0.5

0.2 和 0.3 都转化为二进制后再进行计算
0.2 --> 0.001100110011001100110011001100110011001100110011001101 +
0.3 --> 0.010011001100110011001100110011001100110011001100110011 = 
        0.100000000000000000000000000000000000000000000000000001 // 尾数大于 52 位

而实际取值只取 52 位尾数位，就变成了
0.5 --> 0.1000000000000000000000000000000000000000000000000000

既然 0.1 不是 0.1 了，为什么 0.1 === 0.1

判断时会做精度运算，在转换的过程中超过的精度会自动做凑整处理。

判断

typeof

typeof 是 JavaScript 的一元运算符，它返回一个字符串，表示未经计算的操作数的类型。

被判断类型 结果
Undefined "undefined"
Null "object"
Boolean "boolean"
Number "number"
BigInt "bigint"
String "string"
Symbol "symbol"
Function 对象 "function"
浏览器提供的对象 取决于具体实现
其他任何对象 "object"
