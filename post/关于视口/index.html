关于视口
发表于 2020-11-29|前端知识CSS
字数总计:757|阅读时长: 2 分钟

分类

视口（viewport）包括了布局视口、视觉视口和理想视口。

布局视口

当我们以百分比来指定一个元素的大小时，它的值是基于父元素来计算的。当这个元素是根元素时，它就是基于布局视口（layout viewport）来计算的。

所以，布局视口是网页布局的基准窗口，在 PC 浏览器上，布局视口就等于当前浏览器的窗口大小（不包括 border、margin、滚动条）。

在移动端，布局视口被赋予一个默认值，大部分为 980px，这保证 PC 网页可以在手机浏览器上呈现，但是非常小，用户可以手动对网页进行放大。

可以通过 document.documentElement.clientWidth / clientHeight 来获取布局视口大小。

视觉视口

视觉视口（visual viewport）表示用户通过屏幕真实看到的区域，默认等于当前浏览器的窗口大小（包括滚动条宽度）。

当用户对浏览器进行缩放时，不会改变布局视口的大小，所以页面布局是不变的，但是缩放会改变视觉视口的大小。

例如用户将浏览器窗口放大了 200%，这时浏览器窗口中的 CSS 像素会随着视觉视口的放大而放大，这时一个 CSS 像素会跨越更多的物理像素。

所以，布局视口会限制你的 CSS 布局，而视觉视口决定用户具体能看到什么。

可以通过 window.innerWidth / innerHeight 来获取视觉视口大小。

理想视口

布局视口在移动端展示的效果并不是一个理想的效果，所以理想视口（ideal viewport）就诞生了，用来表示网站页面在移动端展示的理想大小。

如上图，在浏览器调试移动端时页面上给定的像素大小就是理想视口大小，它的单位正是设备独立像素。

可以通过 screen.width / height 来获取理想视口大小。

视口的设置

<meta> 元素表示那些不能由其它 HTML 相关元素之一表示的任何元素数据信息，它可以告诉浏览器如何解析页面。

我们可以借助 <meta> 元素的 viewport 来帮助设置视口、缩放等，从而让移动端得到更好的展示效果。

在苹果的规范中， <meta> 的 viewport 属性有 6 个可选值：

content 值 可能值 描述
width 正整数或 device-width 布局视口的宽度，若为 device-width 则设为理想视口
height 正整数 布局窗口的高度，很少使用
initial-scale [0.0, 10.0] 页面初始缩放比例
minimum-scale [0.0, 10.0] 缩放的最小比例
maximum-scale [0.0, 10.0] 缩放的最大比例
user-scalable yes/no 如果设置为 no，用户将不能放大缩小网页。默认值为 yes
