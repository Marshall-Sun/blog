CSS 元素样式优先级

展开

小结

判断优先级时，首先看是否有 !important，有则为最高权重。

一条匹配规则一般由多个选择器组成，一条规则的特殊性由组成它的选择器的特殊性累加而成。

选择器的特殊性可以分为四个等级：

  1. 行内样式，为 1000
  2. ID 选择器，为 0100
  3. 类选择器、伪类选择器和属性选择器，为 0010
  4. 元素选择器和伪元素选择器，为 0001

规则中选择器的特殊性可叠加，但不会进位。

特殊性值的比较是从左向右排序的，比如说特殊性值为 1000 的优先级比特殊性值为 0999 的规则高。

如果两个规则的特殊性值相等的时候，那么就会根据它们引入的顺序，后出现的规则的优先级最高。

扩展

CSS 的优先级是根据样式声明的特殊性值来判断的。

选择器的特殊性值分为四个等级：

  1. 标签内选择符 x,0,0,0
  2. ID 选择符 0,x,0,0
  3. Class 选择符/属性选择符/伪类选择符 0,0,x,0
  4. 元素和伪元素选择符 0,0,0,x

计算方法：

  1. 每个等级的初始值为 0
  2. 每个等级的叠加为选择器出现的次数相加
  3. 不可进位，比如 0,99,99,99
  4. 依次表示为：0,0,0,0
  5. 每个等级计数之间没关联
  6. 等级判断从左向右，如果某一位数值相同，则判断下一位数值
  7. 如果两个优先级相同，则最后出现的优先级高，!important 也适用
  8. 通配符选择器的特殊性值为：0,0,0,0
  9. 继承样式优先级最低，通配符样式优先级高于继承样式
  10. !important 没有特殊性值，但它的优先级是最高的，可以认为它的特殊性值为 1,0,0,0,0。

  2. 特殊性值越大的声明优先级越高。
  3. 相同特殊性值的声明，根据样式引入的顺序，后声明的规则优先级高（距离元素出现最近的）。
  4. 部分浏览器不考虑字节溢出的进位。

关于长度单位

展开

长度单位有两种：绝对单位和相对单位。

绝对单位

绝对长度单位是固定的值，不建议在已知变化的屏幕尺寸上使用绝对长度单位。

绝对单位 描述
cm centimeters 厘米
mm millimeters 毫米
in inches (1 in = 2.54 cm) 英尺
pc picas (1 pc = 1/6 in) 派卡，印刷行业使用
pt points (1 pt = 1/72 in) 逻辑像素
px pixels (1 px = 1/96 in) 物理像素

注意：物理像素（px）在不同的设备有不同表现。对于低分辨率屏幕，1px 表示一个逻辑像素，对于高分辨率屏幕，1px 表示多个逻辑像素。

相对单位 描述
em 相对于当前元素字体“M”宽度
ex 相对于当前元素字体“M”高度，很少用到
ch 相对于当前元素字体“0”宽度，很少用到
rem 相对于根元素字体“M”宽度
vw 视口宽度 window.innerWidth 的百分比
vh 视口高度 window.innerHeight 的百分比
vmin 取当前 vw 和 vh 中最小值
vmax 取当前 vw 和 vh 中最大值
% 相对于父元素对应长度的百分比

关于盒模型

展开

有两种盒模型：IE 盒模型、W3C 标准盒模型，都分为四部分：内容 content、内填充 padding、边框 border、外边界 margin

IE 盒模型和 W3C 标准盒模型的区别：

  • W3C 标准盒模型：width/height = content
  • IE 盒模型：width/height = content + padding + border
  • width/height 都不包含外边界 margin

在 IE8+ 中使用盒模型的种类可以调节：

  • 如果在 IE6+ 中 DOCTYPE 缺失，默认为 IE 盒模型。
  • 若在页面中声明了 DOCTYPE 类型，所有的浏览器默认为 W3C 盒模型。

视觉格式化模型与 display

展开

背景知识：

  • 块（block），一个抽象的概念，一个块在文档流上占据一个独立的区域，块与块之间在垂直方向上按照顺序依次堆叠。
  • 包含块（containing block），包含其他盒子的块。
  • 盒子（box），一个抽象的概念，由 CSS 引擎根据文档中的内容所创建，主要用于文档元素的定位、布局和格式化等用途。盒子与元素一般情况下一一对应，但有时多个元素会合并生成一个盒子，有时一个元素会生成多个盒子（如匿名盒子）。

元素（element）有两种：块级元素与行内元素：

块级元素 行内元素
如 <div> <p> <h1> 等 如 <a> <span> <code> <img> 等
独占一行，默认宽度填满父元素宽度 相邻的行内元素排列在同一行内，直到一行排不下才会换行，宽度随子元素变化
可以设置 width/height 属性 设置 width/height 无效
可以设置 margin/padding 属性 只能设置 margin-left/right 与 padding-left/right，其它边距效果无效

视觉格式化模型（visual formatting model）是用来处理和显示文档时使用的计算规则，是 CSS 的基础概念之一。

其会根据盒模型将文档中的元素转换为一个个盒子。然后根据盒子的边界来渲染盒子，通常盒子会创建一个包含其子元素的包含块，但是子元素并不由包含块所限制，当子元素跑到包含块的外面时称为溢出（overflow）。

盒子的生成是 CSS 视觉格式化模型的一部分，用于从文档元素生成盒子。盒子有不同的类型，这取决于 display 属性。

常用 display 值 描述
none 不显示
inline （默认值）显示为行内元素
block 显示为块级元素
inline-block 显示为行内块元素
flex 显示为 Flex 容器
list-item 显示为列表
table 显示为块级表格（类似 <table>）
inherit 从父元素继承属性值

布局流与 position

展开

块级元素按照块流动方向（block flow direction）放置，流动规则有三种：普通流、浮动流、定位流。

普通流

在普通流的块格式化上下文中，盒子在垂直方向依次排列；而在行内格式化上下文中，盒子则水平排列。

当盒子具有 position: static/relative，并且 float: none 时，其布局方式为普通流。

浮动流

一个盒子的 float 值不为 none，并且 position: static/relative 时，该盒子为浮动流。

在浮动流中，浮动盒子会浮动到当前行的开始或尾部位置。这会导致普通流中的元素会“流”到浮动盒子的边缘处（被浮动影响），除非元素通过 clear 清除了前面的浮动。

如果 float: left，浮动盒子会流到当前行盒子的开始位置（左侧）；如果 float: right，浮动盒子会流到当前行盒子的尾部位置（右侧）。

不管是左浮动还是右浮动，行盒子都会伸缩以适应浮动盒子的大小。

定位流

元素被移出正常文档流，无预留空间。

position 值 描述
static （默认值）没有定位，元素属于普通流，忽略 top/bottom/left/right/z-index
relative 相对定位，元素属于普通流，相对于正常位置定位
absolute 绝对定位，元素属于定位流，通过第一个非 static 的父元素定位
fixed 固定定位，元素属于定位流，通过浏览器窗口定位
inherit 从父元素继承属性值

关于 BFC

展开

BFC（Block Formatting Context，块级格式化上下文），用于决定块级盒子的子元素布局方式以及浮动相互影响的一个独立区域。

BFC 只对直接子元素产生影响，这间接说明了一个元素不能同时存在两个 BFC 中。

如何创建 BFC

  • 根元素 <html>
  • display: flow-root
  • display: table/table-*（表格元素）
  • display: inline-block（行内块元素）
  • display: flex（弹性元素）
  • position: absolute/fixed（绝对定位元素）
  • overflow 值不为 visible 的块元素
  • float 值不为 none （浮动元素）

BFC 的特性

  • BFC 属于普通流，即内部块级盒会在垂直方向上排列。
  • 计算 BFC 的高度时，浮动元素也会参与计算。
  • BFC 元素不会和它的子元素发生外边距折叠，但子元素间可能发生。
  • 浮动盒子不会和 BFC 重叠。
  • BFC 是一个独立的容器，外面的元素不会影响 BFC 内部。

BFC 的应用

  • 解决父元素高度坍塌
  • 实现自适应布局
  • 解决外边距重叠（边界折叠）

关于 Flex

展开

Flex（Flexible Box，弹性布局）用来为盒模型提供最大的灵活性。

任何一个元素都可以指定为 Flex 布局，包括块级元素和行内元素。设为 Flex 布局以后，子元素的 float/clear/vertical-align 属性失效。

采用 Flex 布局的元素，称为容器（flex container），它的所有子元素（不包括后代）自动成为容器成员，称为项目（flex item）。

容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis），项目默认沿主轴排列。
