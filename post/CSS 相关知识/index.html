关于 CSS 元素样式优先级

展开

小结

判断优先级时，首先看是否有 !important，有则为最高权重。

一条匹配规则一般由多个选择器组成，一条规则的特殊性由组成它的选择器的特殊性累加而成。

选择器的特殊性可以分为四个等级：

  1. 行内样式，为 1000
  2. ID 选择器，为 0100
  3. 类选择器、伪类选择器和属性选择器，为 0010
  4. 元素选择器和伪元素选择器，为 0001

规则中选择器的特殊性可叠加，但不会进位。

特殊性值的比较是从左向右排序的，比如说特殊性值为 1000 的优先级比特殊性值为 0999 的规则高。

如果两个规则的特殊性值相等的时候，那么就会根据它们引入的顺序，后出现的规则的优先级最高。

扩展

CSS 的优先级是根据样式声明的特殊性值来判断的。

选择器的特殊性值分为四个等级：

  1. 标签内选择符 x,0,0,0
  2. ID 选择符 0,x,0,0
  3. Class 选择符/属性选择符/伪类选择符 0,0,x,0
  4. 元素和伪元素选择符 0,0,0,x

计算方法：

  1. 每个等级的初始值为 0
  2. 每个等级的叠加为选择器出现的次数相加
  3. 不可进位，比如 0,99,99,99
  4. 依次表示为：0,0,0,0
  5. 每个等级计数之间没关联
  6. 等级判断从左向右，如果某一位数值相同，则判断下一位数值
  7. 如果两个优先级相同，则最后出现的优先级高，!important 也适用
  8. 通配符选择器的特殊性值为：0,0,0,0
  9. 继承样式优先级最低，通配符样式优先级高于继承样式
  10. !important 没有特殊性值，但它的优先级是最高的，可以认为它的特殊性值为 1,0,0,0,0。

  2. 特殊性值越大的声明优先级越高。
  3. 相同特殊性值的声明，根据样式引入的顺序，后声明的规则优先级高（距离元素出现最近的）。
  4. 部分浏览器不考虑字节溢出的进位。

关于盒模型

展开

有两种盒模型：IE 盒模型 border-box、W3C 标准盒模型 content-box

都分为四部分：内容 content、内填充 padding、边框 border、外边界 margin

IE 盒模型和 W3C 标准盒模型的区别：

  • W3C 标准盒模型：width/height = content，不包含 border 和 padding
  • IE 盒模型：width/height = content + border + padding
  • width/height 都不包含外边界 margin

在 IE8+ 中使用盒模型的种类可以调节：

  • 如果在 IE6+ 中 DOCTYPE 缺失，默认为 IE 盒模型。
  • 若在页面中声明了 DOCTYPE 类型，所有的浏览器默认为 W3C 盒模型。

关于 display 的字段与作用

展开
块级元素 行内元素
独占一行，默认宽度填满父元素宽度 相邻的行内元素排列在同一行内，直到一行排不下才会换行，宽度随子元素变化
可以设置 width/height 属性 设置 width/height 无效
可以设置 margin/padding 属性 只能设置 margin-left/right 与 padding-left/right，其它边距效果无效

常用 display 值 描述
none 不显示
inline （默认值）显示为行内元素
block 显示为块级元素

flex 显示为 Flex 容器
list-item 显示为列表
table 显示为块级表格（类似 <table>）
inherit 从父元素继承属性值

关于 position 的字段与作用

展开
position 值 描述
static （默认值）没有定位，元素正常布局，忽略 top/bottom/left/right/z-index
absolute 绝对定位，元素被移出正常文档流，无预留空间，通过第一个非 static 的父元素定位
fixed 绝对定位，元素被移出正常文档流，无预留空间，通过浏览器窗口定位
relative 相对定位，相对于正常位置定位
inherit 从父元素继承属性值

关于 Flex 的字段与作用

展开

Flex（Flexible Box，弹性布局）用来为盒模型提供最大的灵活性。

任何一个元素都可以指定为 Flex 布局，包括块级元素和行内元素。设为 Flex 布局以后，子元素的 float/clear/vertical-align 属性失效。

采用 Flex 布局的元素，称为容器（flex container），它的所有子元素（不包括后代）自动成为容器成员，称为项目（flex item）。

容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis），项目默认沿主轴排列。
