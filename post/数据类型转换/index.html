强制类型转换

ToBoolean

抽象操作 ToBoolean 负责处理非布尔值到布尔值的强制类型转换。

Boolean() 执行抽象操作 ToBoolean，遵循下列转换规则：

  1. 返回 false

    • undefined
    • null
    • false
    • +0, -0, NaN
    • ""

  2. 其他值返回 true

ToNumber

抽象操作 ToNumber 负责处理非数字到数字的强制类型转换。

Number() 执行抽象操作 ToNumber，遵循下列转换规则：

  1. 如果是 Number 类型，直接返回。

  2. 如果是 Boolean 类型，true 转换为 1，false 转换为 0。

  3. 如果是 null，返回 0。

  4. 如果是 undefined，返回 NaN。

  5. 如果是字符串：

    • 若只包含数字时，转换成十进制的数值，并忽略首位 0，如：Number('0123') -> 123。
    • 包含有效的浮点数，如 1.1 转换成对应的浮点数，如：Number('01.1') -> 1.1。
    • 包含有效的十六进制格式，如 0xf 转换成相同大小的十进制数值，如：Number('0xf') -> 15。
    • 空字符串转换成 0，如：Number('') -> 0。
    • 其他转换成 NaN，如：Number('123x') -> NaN。

  6. 如果是对象，则调用对象的 valueOf() 方法，如果结果是原始类型则直接对该值使用 Number()。

    如果 valueOf() 方法返回的还是对象，则调用 toString() 方法，如果结果是原始类型则直接对该值使用 Number()。一般运行到这里都会得到字符串 "[object Object]"，转换后返回 NaN。

    如果最后还是对象，则报错。

parseInt()

只能将字符串转换成数值，与 Number() 转字符串的区别：

  1. 如果传入字符串是数字开头或者负号开头，则往后取值直到非数字停止，如：

    • parseInt('123x') -> 123
    • parseInt('-023x') -> -23
    • parseInt('-0a') -> -0
    • parseInt('-0x') -> NaN（0x 为十六进制数的开头）

  2. 传入字符串是非数字或者负号开头，则为 NaN，如：

    • parseInt('x123') -> NaN
    • -parseInt('-abc') -> NaN

  3. 空字符串直接返回 NaN，如：parseInt('') -> NaN

  4. parseInt('1.1') -> 1，这也是它和 parseFloat() 的差别

传入值 Number() parseInt()
"" 0 NaN
"true"/"false" 1/0 NaN
"0123" 123 123
"123x" NaN 123
"x123" NaN NaN
"01.1" 1.1 1

Object.prototype.valueOf()

返回指定对象的原始值。

Object 类型 valueOf() 返回值
Array 数组本身
Function 函数本身
Boolean 布尔值本身
Number 数字值本身
Object 对象本身
String 字符串本身
Date 从 1970 年 1 月 1 日午夜开始计的毫秒数

注意：Math 和 Error 对象没有 valueOf() 方法。

可以创建一个覆盖自定义对象的 valueOf()，但其必须不能传入参数。

ToString

抽象操作 ToString 负责处理非字符串到字符串的强制类型转换。

String() 执行抽象操作 ToString，遵循下列转换规则：

  1. 如果值是 null，返回 "null"。
  2. 如果值是 undefined，返回 "undefined"。
  3. 调用 toString() 方法并返回相应的结果。

toString()

返回一个表示该对象的字符串，当对象表示为文本值或以期望的字符串方式被引用时，toString() 被自动调用。

valueOf() 与 toString() 的区别

valueOf() 偏向于运算，toString() 偏向于显示。

  1. 在进行对象转换时，将优先调用 toString()，如若没有重写 toString()，将调用 valueOf()；如果两个方法都没有重写，则按 Object 的toString() 输出。
  2. 在 ToString 时，将优先调用 toString 方法，ToNumber 时优先调用 valueOf()。
  3. 使用运算操作符的情况下，valueOf() 的优先级高于 toString()。

自动类型转换是以强制类型转换为基础的，遇到以下三种情况时，JavaScript 会自动转换数据类型，即转换是自动完成的，用户不可见。

ToNumber

JavaScript 遇到预期为数值的地方，就会将参数值自动转换为数值。系统内部会自动调用 Number()。

除了加法运算符 + 有可能把运算子转为字符串，其他运算符都会把运算子自动转成数值。

一元运算符也会把运算子转成数值。

ToString

JavaScript 遇到预期为字符串的地方，就会将非字符串的值自动转为字符串。具体规则是，先将复合类型的值转为原始类型的值，再将原始类型的值转为字符串。

字符串的自动转换，主要发生在字符串的加法运算时。当一个值为字符串，另一个值为非字符串，则后者转为字符串。

相等运算符 == 运算过程

  1. 如果 y 不是正常值，中断执行。
  2. 如果两者类型相同，返回 x === y。
  3. 如果两者一个是 null，另一个是 undefined，返回 true。
  4. 如果 x 是数值，y 是字符串，返回 x == ToNumber(y)。
  5. 如果 x 是字符串，y 是数值，返回 ToNumber(x) == y。
  6. 如果 x 是布尔值，返回 ToNumber(x) == y。
  7. 如果 y 是布尔值，返回 x == ToNumber(y)。
  8. 如果 x 是字符串/数值/Symbol，y 是对象，返回 x == ToPrimitive(y)。
  9. 如果 x 是对象，y 是字符串/数值/Symbol，返回 ToPrimitive(x) == y。
  10. 返回 false。
