关于懒加载
发表于 2020-11-29| 更新于 2020-11-29|前端知识JavaScript
字数总计:434|阅读时长: 1 分钟

定义

懒加载（lazy loading，延迟加载）一般应用于图片。当访问一个页面的时候，先把图片元素或是其他元素的背景图片路径设置为一张占位图，只有当图片出现在浏览器的可视区域内时，才设置图片真正的路径，让图片显示出来。

本网站就广泛应用了懒加载技术。

意义

  • 能提升用户的体验。不妨设想下，用户打开像手机淘宝长页面的时候，如果页面上所有的图片都需要加载，由于图片数目较大，等待时间很长，用户难免会心生抱怨，这就严重影响用户体验。
  • 减少无效资源的加载。这样能明显减少了服务器的压力和流量，也能够减小浏览器的负担。
  • 防止并发加载的资源过多，阻塞 JavaScript 的加载，影响网站的正常使用。

实现方法

可视区加载，即仅加载用户可以看到的区域。

先将图片的 src 属性设置为占位图，而真实的图片 URL 设置在 data-src 中。监听 scroll 事件，当图片节点出现在视图中时，就将 src 的值设置为 data-src 的值。

一般会在距用户看到某图片前一定距离就开始加载，这样能保证用户拉下时正好能看到图片。
