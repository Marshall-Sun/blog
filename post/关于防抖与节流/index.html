关于防抖与节流
发表于 2020-11-29|前端知识JavaScript
字数总计:802|阅读时长: 2 分钟

防抖

定义

小明军训，教官发令：向左转！向右转！向后转！大家都照着做，唯有小明不动。教官大声斥问他为啥不听指挥？小明说，我准备等你想好到底往哪个方向转，我再转。

这就是防抖（debounce）：防止抖动，以免把一次事件误认为多次。N 秒内函数不会执行，若这段时间内高频事件再次被触发，则重新计时。

我们可以用极端情况理解它：如果 N 足够大，并且期间一直有动作触发，那么回调就永远不会执行。放在上文的语境里就是，只有教官最后一次发号后，小明才会转。

代码实现

  1. 设置定时器。
  2. 设置一个闭包，返回一个方法。
  3. 如果被反复调用，清空前面的定时器，再重新设置一遍。

节流

定义

学生上自习课，班主任 N 秒内巡视一次，这段时间内多少人吵闹都可以，但是在间隔点上吵闹的学生就会被班主任点名批评。

这就是节流（throttle）：控制水的流量，即事件发生的频率。指定一个 N 秒的间隔时间，不管高频事件触发多少次，只会在间隔点上执行。

我们也可以用极端情况理解它：如果期间一直有动作触发，那么就等于函数隔 N 秒定期执行。放在上文的语境里就是，如果教室里一直吵闹，班主任每隔 N 秒都会点名批评一个不幸的学生。

代码实现

有两种实现方法，区别如下：

方法 开始触发时 停止触发后
定时器实现 N 秒后执行 继续执行一次事件
时间戳实现 立刻执行 不再执行事件

定时器实现

  1. 设置一个标记。
  2. 设置一个闭包，返回一个方法。
  3. 如果重复执行的时候，标记不为 null，则阻止本次运行。
  4. 如果定时器执行完了，设置这个标记为 null，允许下一次执行。

使用场景

防抖：多次事件合为一次触发

  • 防止表单多次提交。
  • 输入框连续输入，减少搜索发起次数。
  • 文本编辑器实时保存，无操作一段时间后自动保存。
  • 窗口缩放，等待缩放完毕后，才重新计算 DOM 尺寸。

节流：多次事件按时间平均触发

  • 鼠标拖拽事件。
  • 鼠标滚动事件，如页面无限加载时下拉触发加载。
