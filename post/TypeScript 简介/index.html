TypeScript 简介
发表于 2020-12-05| 更新于 2020-12-05|前端知识TypeScript
字数总计:627|阅读时长: 1 分钟

定义

TypeScript 是 JavaScript 的一个超集，可以编译成纯 JavaScript。它主要提供了类型系统和对 ES6 的支持。

它由 Microsoft 开发，代码开源于 GitHub 上。

优缺点

增加了代码的可读性和可维护性

  • 类型系统实际上是最好的文档，大部分的函数看看类型的定义就可以知道如何使用了。
  • 可以在编译阶段就发现大部分错误，这总比在运行时候出错好。
  • 增强了编辑器和 IDE 的功能，包括代码补全、接口提示、跳转到定义、代码重构等。

包容

  • TypeScript 是 JavaScript 的超集，.js 文件可以直接重命名为 .ts 即可。
  • 即使不显式的定义类型，也能够自动做出类型推论。
  • TypeScript 的类型系统是图灵完备的，可以定义从简单到复杂的几乎一切类型。
  • 即使 TypeScript 编译报错，也可以生成 JavaScript 文件。
  • 兼容第三方库，即使第三方库不是用 TypeScript 写的，也可以编写单独的类型文件供 TypeScript 读取。

拥有活跃的社区

  • 大部分第三方库都有提供给 TypeScript 的类型定义文件。
  • Angular、Vue、VS Code、Ant Design 等等耳熟能详的项目都是使用 TypeScript 编写的。
  • TypeScript 拥抱了 ES6 规范，支持 ESNext 草案中处于第三阶状态（Stage 3）的特性。

缺点

  • 有一定的学习成本，需要理解接口（Interfaces）、泛型（Generics）、类（Classes）、枚举类型（Enums）等概念。
  • 短期可能会增加一些开发成本，毕竟要多写一些类型的定义，不过对于一个需要长期维护的项目，TypeScript 能够减少其维护成本。
  • 集成到构建流程需要一些工作量。
  • 可能和一些库结合的不是很完美。

编辑器

TypeScript 最大的优势之一便是增强了编辑器和 IDE 的功能，包括代码补全、接口提示、跳转到定义、重构等。

主流的编辑器都支持 TypeScript，这里强烈推荐使用 Visual Studio Code。

它是一款跨终端的轻量级编辑器，原生支持 TypeScript。它本身也是用 TypeScript 编写的，源代码开源于 Github 上。

在 Stack Overflow 的 2019 年开发者调查中，Visual Studio Code 成为了最受欢迎的开发工具。

本网站全程使用 Visual Studio Code 编写，包括网站本身与网站内的所有文章。
