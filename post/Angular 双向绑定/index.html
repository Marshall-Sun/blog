Angular 双向绑定
发表于 2020-12-03|前端知识Angular
字数总计:1.3k|阅读时长: 4 分钟

数据绑定

数据绑定是一种机制，用来协调用户可见的内容，特别是应用数据的值。虽然也可以手动从 HTML 中推送或拉取这些值，但是如果将这些任务转交给绑定框架，应用就会更易于编写、阅读和维护。开发者只需声明数据源和目标 HTML 元素之间的绑定关系就可以了，Angular 会完成其余的工作。

Angular 提供了多种数据绑定方式。绑定类型可以分为三类，按数据流的方向分为：

分类 绑定类型 语法
单向，从数据源到视图 插值 [target]="expression"
bind-target="expression"
单向，从视图到数据源 事件 (target)="statement"
on-target="statement"
双向，视图到数据源到视图 双向 [(target)]="expression"
bindon-target="expression"

双向绑定 [(...)]

双向绑定提供了一种在组件类及其模板之间共享数据的方式。

双向绑定会做两件事：

  • 设置特定的元素属性。
  • 监听元素的变更事件。

Angular 为此提供了一种特殊的双向数据绑定语法 [()]。该元素具有名为 x 的可设置属性和名为 xChange 的相应事件。

脏检查

在上文中，Angular 到底应该怎样得知 resize() 执行后是否应该刷新 UI，应该更新哪个 DOM 元素？

Angular 必须去挨个检查这些元素对应绑定表达式的值是否有被改变。这就是脏检查机制：

脏检查机制（Dirty checking，脏检测）是在数据变化中，确保 Angular 已经收集到所有的数据变化，真正要进行界面的更新，仍然需要 Angular 内置的视图渲染和更新渲染的相关机制。

与 React 对比

同样的道理，在 React 中也有一套系统，在数据变化时确保收集到数据的变化，这就是 Virtual DOM 机制：

React 依赖 props、state 以及内置提供的一些方法，直接修改数据对象属性来触发数据变化通知。

虽然脏检查和 Virtual DOM 是在不同阶段的不同概念，但是数据变化和对数据变化做出界面响应，这两个阶段是它们内在最本质的特性。

因此，这两个阶段是不可分离的，把这两个概念放在一起也可以笼统总结出它们界面响应机制的区别。

脏检查原理

在 Angular 中是谁来通知数据即将变化的呢？在 AngularJS 中是由代码 $scope.$apply() 或者 $scope.$digest 触发，而 Angular 接入了 ZoneJS，由它监听了 Angular 所有的异步事件。

Angular 收到通知后，开始使用脏检查执行变化检测。其实就是存储所有变量的值，每当可能有变量发生变化需要检查时，就将所有变量的旧值跟新值进行比较，如果旧值跟新值不相等就说明变化了，需要更新对应视图。

运行效率

脏检查效率不高，但是也谈不上有多慢。简单的数字或字符串比较能有多慢呢？十几个表达式的脏检查可以直接忽略不计；上百个也可以接受；成百上千个就有很大问题了。

因此，绑定大量表达式时应该注意：

  • 主要：尽量减少绑定表达式的个数（可使用单次绑定和 ng-if）
  • 表达式（以及表达式所调用的函数）中少写太过复杂的逻辑。
  • 不要连接太长的 filter。（往往 filter 里都会遍历并且生成新数组）
  • 不要访问 DOM 元素。
