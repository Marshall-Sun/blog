关于响应式布局
发表于 2020-11-29|前端知识CSS
字数总计:869|阅读时长: 2 分钟

背景知识

  • 关于视口
  • CSS 长度单位

定义

响应式网页设计（Responsive web design，RWD，响应式布局）指的是同一页面在不同屏幕尺寸下有不同的布局。为了适配移动端，传统的开发方式是自适应布局，即 PC 端开发一套，移动端再开发一套，而使用响应式布局只要开发一套就够。

解决方案

媒体查询

CSS3 中引入了媒体查询（Media queries），它可以让我们针对不同的媒体类型定义不同的样式。当重置浏览器窗口大小的过程中，页面也会根据浏览器的宽度和高度重新渲染页面。

百分比布局

除了用 px 结合媒体查询实现响应式布局外，也可以通过百分比单位 % 来实现响应式。

我们知道，% 是相对于父元素对应长度的百分比。但是到底是相对于父元素的哪部分？

子元素百分比属性 相对于父元素值
width width
height height
padding width
margin width
border 不能设置百分比
border-radius 水平半轴相对于子元素的 width
垂直半轴相对于子元素的 height

因此可以看出百分比布局的缺点：

  • 计算相对困难，如果我们要定义一个元素的宽度和高度，那么须换算成百分比单位。
  • 各个属性中如果使用百分比，相对父元素的属性并不是唯一的。这可能使布局问题变得复杂。

rem 布局

rem 是 CSS3 新增的单位，并且移动端的支持度很高。rem 单位是相对于根元素字体宽度变化的，这相当于提供了一个基准。

当页面大小变化时，只需要改变 font-size 的值，那么相关元素的大小也会发生响应变化。因此，rem 布局只需要根据视图容器的大小改变 font-size 即可。

flexible 方案是阿里巴巴早期开源的一个移动端适配解决方案，它的核心代码非常简单，通过设置 document.documentElement.style.fontSize 就可以统一整个页面的布局标准。

但是 rem 布局也具有缺点：

在响应式布局中，必须通过 JavaScript 来动态控制根元素 font-size 的大小，也就是说 CSS 和 JavaScript 有一定的耦合性，且必须将改变 font-size 的代码放在 CSS 之前。

视口单位

vw/vh 也是 CSS3 新增的单位，是视觉视口宽度 window.innerWidth/innerHeight 的百分比。

绝大多数的浏览器支持视口单位，但是还有少数浏览器不支持。如果需要兼容所有浏览器，不推荐使用视口单位。
