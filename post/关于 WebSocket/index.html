关于 WebSocket
发表于 2020-12-02|前端知识计算机网络
字数总计:581|阅读时长: 1 分钟

定义

如果要实现股票的实时行情，或者获取火车票的剩余票数等，就需要客户端和服务器之间反复地进行 HTTP 通讯，客户端不断地发送 GET 请求，去获取当前的实时数据，这就是轮询（Polling）。

为了彻底解决服务器主动向客户端发送数据的问题，HTML5 新增了一种应用层协议 WebSocket，它是一种客户端与服务器进行全双工通讯的网络技术。

它有几个特点：

  • 基于 TCP 传输协议，并复用 HTTP 的握手通道。
  • 与 HTTP 协议有着良好的兼容性，默认端口也是 80 和 443。
  • 双向平等对话，即客户端与服务器可互相主动推送信息。
  • 使用简易，数据格式轻量，性能开销小，通信高效。
  • 没有同源限制。

WebSocket 的协议标识符是 ws（如果进行 SSL/TLS 加密，则为 wss），服务器网址就是 URL：

ws://example.com:80/some/path

在 HTTP 连接期间，浏览器（使用 HTTP header）问服务器：“你支持 WebSocket 吗？”如果服务器回复说“我支持”，那么通信就由之前的 HTTP 协议转变成 WebSocket 协议继续进行。

因此，如果要建立一个 WebSocket 连接，客户端需要与服务器先建立 HTTP 连接。

断开连接

相比于传统 HTTP 的每次请求应答都要客户端与服务器建立连接，WebSocket 是一种长连接的模式，类似 Connection: keep-alive。

具体什么意思呢？就是一旦 WebSocket 连接建立后，除非有一端主动断开连接，否则每次数据传输都可以复用已建立的连接，不需要重新三次握手四次挥手。
