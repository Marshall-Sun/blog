Leetcode/79.单词搜索
发表于 2020-11-08| 更新于 2020-11-08
字数总计:780|阅读时长: 3 分钟

问题描述

给定一个二维网格和一个单词，找出该单词是否存在于网格中。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

示例:

board =
[
  ['A','B','C','E'],
  ['S','F','C','S'],
  ['A','D','E','E']
]

给定 word = "ABCCED", 返回 true
给定 word = "SEE", 返回 true
给定 word = "ABCB", 返回 false

提示：

  • board 和 word 中只包含大写和小写英文字母。
  • 1 <= board.length <= 200
  • 1 <= board[i].length <= 200
  • 1 <= word.length <= 10^3

解法

记录访问过的点

用一个二维矩阵 used，记录已经访问过的点，下次再选择访问这个点，就直接返回 false。

为什么要回溯

有的选择是错的，选择它，不能构成目标路径，继续选，选不出结果。所以要撤销这个选择，去尝试别的选择。

如果第一个递归调用返回 false，就会执行||后面的下一个递归（其实这里有隐式的回溯，因为我们回到了 [row,col] 这个坐标，选择另一个坐标 [row-1,col] 进行递归）。

只要其中有一个返回 true，||后面的逻辑就不会执行，即找到了就不用再递归搜索了，利用了||的短路效应，起到剪枝的效果。

canFind 代表：基于当前选择的点，四个搜索方向上，能否找到剩下的字符的路径。

如果不能，说明当前点不是正确的选择，不仅要返回false，还要在 used 中把当前点恢复为未访问状态。

因为，基于目前的路径，选它是不对的，但基于别的路径，选它，有可能是对的。

恢复为未访问状态，它在后续就能正常被访问了。

递归的结束条件

在递归中我们设置了所有返回 false 的情况。当指针越界，此时已经考察完单词字符。

意味着，在这个递归中，为一个个字符选点，始终没有返回过 false，这些字符都选到了正确的点。

所以指针越界就可以返回真了。
