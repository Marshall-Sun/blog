Promise 什么作用呢？

Promise 是一个对象，它代表了未来将要发生的事件，用来表示一个异步操作的最终完成或者失败。

它是一个承诺：把某件事交给某个人，这个人做完后，不管失败还是成功，都会给你一个回应，这样就可以安心将事情交给它了。

如果用 Promise 就要与 then、catch、finally 结合使用，这就可能会导致代码冗余，比如说有很多 then 嵌套，这样原来的语义就会变得不清楚。

async 什么作用呢？

async 函数是生成器函数的语法糖，它要和 await 语句结合才能使用。

await 语句其实就是 then 的语法糖，它和 then 一样，返回值都是一个 Promise 对象。

如果用 async 函数的话：

  • 可以用同步的语义编写异步代码
  • 错误处理直接使用 try...catch 语句就可以解决问题，简单明了，而不是像 Promise 一样使用 catch() 函数

说下 JavaScript 的数据类型种类

八种，String, Number, Boolean, Object, Undefined, Null, Symbol, BigInt。

栈储存着原始数据类型，共七种：String, Number, Boolean, Undefined, Null, Symbol, BigInt。存储占据空间小、大小固定，属于被频繁使用数据，所以放入栈中存储。

堆储存着引用数据类型，共一种：Object。存储占据空间大、大小不固定。如果存储在栈中，将会影响程序运行的性能；引用数据类型在栈中存储了指针，该指针指向堆中该实体的起始地址。当解释器寻找引用值时，会首先检索其在栈中的地址，取得地址后从堆中获得实体。

Symbol 代表创建后独一无二且不可变的数据类型，它主要是为了解决可能的全局变量冲突。

BigInt 是一种数字类型数据，它可以表示任意精度的整数，故可以安全地存储和操作超出 Number 范围 [-(2^53 - 1), 2^53 - 1] 的整数。

Undefined 表示一个变量自然的、最原始的状态值。(值不存在)

Null 则表示一个变量被人为的设置为空对象，而不是原始状态。(值存在但是为空，没有意义)

讲下 ECMAScript

Ecma 国际（Ecma International）是一家制定并维护计算机各种标准的国际组织。前名为欧洲计算机制造商协会（European Computer Manufacturers Association，ECMA）。

TC39 是 Ecma 国际的一个部门，专门负责制定 ECMA 第 262 号标准，标准代号为 ECMA-262，这个标准就是 ECMAScript。

JavaScript 则是一种通用脚本编程语言，它遵循了 ECMAScript 标准。换句话说，JavaScript 是 ECMAScript 的一种方言。

版本 发布时间 名称 与前版本的差异
1 1997 ES1 首版
5.1 2011 ES5.1 形式与国际标准 ISO/IEC 16262:2011 完全一致
6 2015 ES2015（ES6） 类、模块化、箭头函数、函数参数默认值、模板字符串、解构赋值、展开运算符、剩余参数、对象属性简写、Promise、let 与 const
7 2016 ES2016（ES7） Array.prototype.includes()、幂运算符

10 2019 ES2019（ES10） 新增了一系列接口
11 2020 ES2020（ES11） 可选链式调用、空值合并、动态引入、全局对象

说下 var、let 与 const

ES2015（ES6）之前，声明变量的方法只有 var。let 与 const 的加入引入了块级作用域的概念。

它们的区别：

  • let 和 const 声明形成块作用域。
  • 同一作用域下 let 和 const 不能声明同名变量，而 var 可以。
  • var 声明的变量会挂载在 window 上，而 let 和 const 声明的变量不会。
  • var 声明变量存在变量提升，let 和 const 不存在变量提升，如果在声明之前使用这些变量就会报错，这就是暂时性死区（temporal dead zone）。
  • const 一旦声明必须赋值，不能使用 null 占位。原始数据类型声明后不能再修改，引用数据类型可以修改其属性。
