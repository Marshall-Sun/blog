知道 Virtual DOM 吗，它和真实的 DOM 有什么区别

JSX 是 JavaScript 的一个语法扩展，它最终会被转译成 React.createElement。它返回的就是虚拟 DOM。

为什么要这样？因为 DOM 操作十分昂贵，大多数时候操作 DOM 是很耗性能的，这是因为 JavaScript 对象的属性（properties）映射到的是 DOM 对象的特性（attributes）上。

我们操作 JavaScript 对象只是修改一个对象，而修改 DOM 对象时还会修改对象的 attributes，那么性能就消耗在 JavaScript 对象和 DOM 对象的转换和同步上。并且，操作 DOM 还会影响页面的渲染，进而会再一次降低了性能。

我们可以通过实现高效的 diff 算法，对比出操作前后有差异的 DOM 节点，然后只对更改了的 DOM 进行更新。这样可以大大减少没有必要的 DOM 操作，进而大幅提升性能。

React 的 diff 算法其实很简单：对比当前真实 DOM 和虚拟 DOM，在对比过程中直接更新真实 DOM。并且这个比对是同级比对，当发现这一级的 DOM 不同时，会直接更新该节点及其所有子节点。这种 diff 策略实际上是出于性能上的取舍。首先，DOM 操作很少出现跨层级的情况，所以只需要同级比对就可以满足大多数情况，也就没有必要对比所有的 DOM 节点，因为那样代价太高。

脏检查

Angular 必须去挨个检查这些元素对应绑定表达式的值是否有被改变。这就是脏检查机制：

脏检查机制（Dirty checking，脏检测）是在数据变化中，确保 Angular 已经收集到所有的数据变化，真正要进行界面的更新，仍然需要 Angular 内置的视图渲染和更新渲染的相关机制。

与 React 对比

同样的道理，在 React 中也有一套系统，在数据变化时确保收集到数据的变化，这就是 Virtual DOM 机制：

React 依赖 props、state 以及内置提供的一些方法，直接修改数据对象属性来触发数据变化通知。

虽然脏检查和 Virtual DOM 是在不同阶段的不同概念，但是数据变化和对数据变化做出界面响应，这两个阶段是它们内在最本质的特性。

因此，这两个阶段是不可分离的，把这两个概念放在一起也可以笼统总结出它们界面响应机制的区别。

脏检查原理

在 Angular 中是谁来通知数据即将变化的呢？Angular 接入了 ZoneJS，由它监听了 Angular 所有的异步事件。

Angular 收到通知后，开始使用脏检查执行变化检测。其实就是存储所有变量的值，每当可能有变量发生变化需要检查时，就将所有变量的旧值跟新值进行比较，如果旧值跟新值不相等就说明变化了，需要更新对应视图。

HTTP协议有哪几种请求类型

RESTful API 是目前比较成熟的一套 API 设计理论，它基于表现层状态转移（Representational State Transfer，REST）原则。

它的主要内容是：

  • 看 URL 就知道要什么

  • 看 HTTP 请求方法就知道干什么

  • 看 HTTP 状态码就知道结果如何

  • GET：从服务器取出资源（一项或多项）。

  • POST：在服务器新建一个资源。

  • PUT：在服务器更新资源（客户端提供改变后的完整资源）。

  • PATCH：在服务器更新资源（客户端提供改变的属性）。

  • DELETE：从服务器删除资源。
