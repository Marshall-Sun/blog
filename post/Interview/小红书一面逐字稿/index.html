在项目中有没有涉及到数据通信，就是客户端和服务器的通信？用什么方式实现？

数据通信在前端这边要做的主要有两件事：

  1. 向服务器发起 HTTP 请求

    React 项目

    React 项目接入了 Firebase 的实时数据库，然后 Firebase 是有相关的接口，就根据官方文档的相关说明直接调用接口，就可以发起 HTTP 请求。

    因为 Firebase 是被 Google 收购了，所以 Firebase 的官方文档现在是 Google 在维护，文档质量挺高的。

    除了使用 Firebase 的接口，在 React 中还可以使用 fetch，这是 JavaScript 官方提供的接口。

    但是在我们的这个项目中大部分都是用了 Firebase 的接口。

    Angular 项目

    Angular 提供了 HTTPClientModule 这个模块，调用它主要有三个步骤：

    1. 从 @angular/common/http 中导入 HttpClient 这个服务类
    2. 进行依赖注入，就是在 constructor 中对导入的 HttpClient 进行声明
    3. 根据需要发起的 HTTP 请求方式，调用其对应的方法。比如要发起 GET 请求，就调用它的 get() 方法；要发起 POST 请求，就调用它的 post() 方法

  2. 通过异步编程，在服务器返回的 HTTP 响应报文中获得数据。

    在我们的 Angular 项目中，调用接口之后会返回一个 Observable 对象，可以使用它的 subscribe 回调函数来获得数据；或者调用它的 toPromise() 方法，将它转换成 Promise 对象。

    在我们的 React 项目中，调用接口会直接返回一个 Promise 对象，然后就可以用 Promise.then() 来获得数据，这也是我一开始的做法。

    之后我使用了 async 函数结合 await 语句对异步编程这部分的代码进行了改进，使代码更为简洁。

Promise 什么作用呢？

Promise 是一个对象，它代表了未来将要发生的事件，用来表示一个异步操作的最终完成或者失败。

它是一个承诺：把某件事交给某个人，这个人做完后，不管失败还是成功，都会给你一个回应，这样就可以安心将事情交给它了。

如果用 Promise 就要与 then、catch、finally 结合使用，这就可能会导致代码冗余，比如说有很多 then 嵌套，这样原来的语义就会变得不清楚。

async 什么作用呢？

async 函数是生成器函数的语法糖，它要和 await 语句结合才能使用。

await 语句其实就是 then 的语法糖，它和 then 一样，返回值都是一个 Promise 对象。

如果用 async 函数的话：

  • 可以用同步的语义编写异步代码
  • 错误处理直接使用 try...catch 语句就可以解决问题，简单明了，而不是像 Promise 一样使用 catch() 函数

开发的时候有没有遇到跨域问题？

有。跨域问题在我开发过程中是比较普遍的，这是因为有同源策略。

如果两个 URL 的协议、域名、端口三者一致，那么它们同源。如果两个 URL 不同源，那么会有三方面的限制：

  1. Cookie、LocalStorage 无法读取。
  2. DOM 无法获得。
  3. 异步网络请求不能发送。

这就是跨域问题，虽然这些限制是必要的，但是在开发过程中也会受到影响：

  • 前后端服务器分开部署，导致端口不一致。
  • 本地预览项目时，请求了在线服务器的数据。
  • 调取了第三方平台的接口。

解决方案

跨域的解决方案主要有四种：JSONP、CORS、WebSocket、代理。

我们的 Angular 项目主要使用了代理的方法来解决跨域问题。

因为跨域主要是浏览器跟服务器出现的问题，如果服务器跟服务器请求就不存在跨域的问题。

Angular 内置了代理的功能，底层是基于 http-proxy-middleware 的，只要编写 proxy-config，它就会使用正则表达式来根据 HTTP 请求的 URL 来转发到对应地址，实现跨域。

有没有了解过 Webpack 一些配置？

Webpack 是一个用于现代 JavaScript 应用程序的静态模块打包工具。

我们项目的打包本质上都是用到了 Webpack，不过基本上都是通过 CLI 直接生成打包文件的，具体的进阶配置我本人没有专门研究过。

开发时常用什么 Git 命令？

  • git add：将工作区修改的内容提交到暂存区。
  • git commit：将暂存区的内容提交到本地仓库。
  • git branch：查看编辑 Git 的分支。
  • git push：上传本地仓库分支到远程仓库分支，实现同步。
  • git pull：从远程获取代码并合并本地的版本，实现同步。
  • git merge：把两个分支合并起来，产生一个额外的合并提交。
  • git rebase：将两个分支融合成一个线性的分支，产生线性的项目历史记录。
  • git reset：把当前分支指向另一个位置。
  • git revert：用一个新提交来消除一个历史提交所做的所有修改。

Angular 的异步路由守卫是解决什么问题？

异步路由守卫，就是当用户访问某一个 URL 的时候，前端要判断 URL 是否应该被当前用户访问。

因为不同的用户有不同的权限，比如包括管理员，学生还有教师，学生肯定是不能访问课程管理界面。

所以就要判断用户是否有访问权限，如果不能访问就要提醒用户，并且跳转到首页。

Angular 在路由守卫这方面提供了一些钩子函数，我们的项目中主要使用了 CanActivate，就是当导航到某路由时，判断是否能被访问并做出对应操作。

我的具体做法是：

  1. 在 CanActivate 函数前加上 async，让它成为 async 函数，这样就可以在其中实现异步编程
  2. 把用户身份与当前模块信息发送给后端，让后端来判断当前模块是否可以被此用户访问
  3. 然后从后端拿到响应数据，并进行相关逻辑处理
  4. 最后在 CanActivate 中 return 一个布尔值，代表是否能够被访问

如何实现响应式布局？

我们几个项目中都有用到响应式布局，主要是用媒体查询和三个相对长度单位来实现。

媒体查询就是针对不同的媒体类型应用不同的样式，它设置几个节点，在节点区间之间就会运用对应的 CSS 样式，这样就可以同一套 CSS 代码，适应不同屏幕的尺寸。

除了媒体查询，也要结合三个相对长度单位：百分比、rem、视口。

  • 首先是百分比，子元素的长度如果设置百分比，都是相对于父元素的长度。

  • rem 是相对于根元素字体宽度变化的，这相当于提供了一个基准。因此如果修改根元素字体宽度，就可以一次性统一调整所有使用 rem 单位的 CSS 长度。

  • 视口，就是 vw/vh，是视觉视口宽度 window.innerWidth/innerHeight 的百分比。

有了这三个相对长度单位，再结合媒体查询，就可以方便的实现响应式布局。

Express 使用到了吗，怎么传给前端数据？

Express 我主要用来对 MySQL 做读写。我们学校有一些课后作业需要用到前后端交互，我主要是用这个来做后端。

Express 框架的核心是对 HTTP 模块的再包装，就等于在 HTTP 模块之上，加了一个中间层。

它的使用主要有三个步骤：

  1. 首先启动 Express，开始监听一个端口。
  2. 通过 use() 方法来注册中间件，实现简单的路由。
  3. 通过路由来展示一个 URL 请求，然后前端发起 HTTP 请求，返回 HTTP 响应报文。

AntV 是用来做什么的？

AntV 就是 Ant Design 的一部分，它主要用于数据可视化，就类似于 Echarts。

我们主要用它来将数据绘制成折线图，它里面提供了一个折线图的方法，我们就调用了它，然后绘制了折线图。

做页面有没有涉及到表单的操作？

有，主要是用了 Angular 的响应式表单。

它提供了一种模型驱动的方式来处理表单输入，其中的值会随时间而变化。

在使用响应式表单时，一个 FormControl 类的实例对应于一个表单控件。

在使用时，通过将控件的实例赋值给属性，之后可以通过监听它来跟踪表单控件的值和状态。

通过使用 FormControl 控件的 value 属性，可以获得当前表单控件的一份数据值拷贝；通过 setValue 方法则可以更新表单的控件值。

当时技术选型的时候，前端框架是怎么选择 Angular 的？

当时我进实验室的时候，我们导师想用一个大家最熟悉的框架来做这个新项目，Angular 当时会用的人比较多，有七八个研究生都对它比较熟悉。

而且 Angular 当时是 Google 在大力推的，它的中文文档也是 Google 在维护，文档质量非常高，所以我们就自然而然的使用了 Angular。

React 和 Vue 当时我都不是很了解，所以我也只是先学了 Angular，就用了 Angular。

React 的 Hook 有使用过吗？

Hook 是 React 16.8 的新增特性。它可以在不编写 class 的情况下使用 state 以及其他的 React 特性。

在这之前，组件的标准写法是类，Hook 的出现使得组件的最佳写法变成了函数。

Hook 的设计目的，就是完全不使用类就能写出一个全功能的函数组件。

Hook 就是钩子，在函数组件中，如果需要外部功能和副作用，就用钩子把外部代码钩进来。

Hook 有很多，我基本上只用到了 useState()，它可以为函数组件引入状态。纯函数不能有状态，所以把状态放在钩子里面。

它返回一个数组，包含两个成员：

  • 第一个成员是一个变量，指向状态的当前值。
  • 第二个成员是一个函数，用来更新状态。

父子组件之间怎么通信？

React

父组件向子组件通信

React 是单向数据流，父组件数据一般通过 props 传递到子组件。

不能在子组件中修改 props 的值，如果父组件的某个 props 改变，React 会重新渲染所有子节点。

子组件向父组件通信

子组件向父组件通信也是通过 props。父组件通过 props 传递回调函数给子组件，子组件调用回调函数，传入参数给父组件实现通信。

Angular

@Input() 和 @Output() 是 Angular 提供的两个装饰器，它们为子组件提供了一种与其父组件通信的方法。

父组件向子组件通信

@Input() 允许父组件更新子组件中的数据，主要有三个步骤：

  1. 在子组件中导入 Input
  2. 在子组件中用 @Input() 装饰需要被子组件读取的属性
  3. 在父组件的 HTML 中将这个属性与子组件绑定

子组件向父组件通信

@Output() 允许子组件向父组件发送数据，主要有四个步骤：

  1. 在子组件中导入 Output 和 EventEmitter，EventEmitter 是 Angular 中用来发出自定义事件的类。
  2. 在子组件中用 @Output() 装饰需要被父组件读取的属性
  3. 在子组件的 HTML 中绑定这个属性
  4. 在父组件的 HTML 中将输入的事件与父组件内部方法绑定，实现传入参数

写 React 的时候有没有做状态管理，比如说 Redux？

为了弥补 React 中麻烦的兄弟组件通讯方式，Redux 建立起一个中央机制，方便各组件之间的通讯。

当 React 组件需要调用数据的时候，它就发起 Actions 并通知管理员（Store），Store 就去查找相关资料（Reducers），拿到返回的信息给组件。

我写 React 主要是单页应用，基本上只用到了父组件向子组件通信，不需要复杂的组件通讯，所以对 Redux 相对了解并不多。
