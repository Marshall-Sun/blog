改变 this 的指向

以下几种方法可以改变 this 的指向：

  1. 使用 ES6 的箭头函数
  2. 在函数内部使用 _this = this
  3. 使用 apply/call/bind
  4. new 实例化一个对象

箭头函数

众所周知，ES6 的箭头函数是可以避免 ES5 中使用 this 的坑的。

箭头函数的语法比函数表达式更简洁，并且没有自己的 this, prototype, super, arguments, new.target。箭头函数表达式更适用于那些本来需要匿名函数的地方，并且它不能用作构造函数。

箭头函数的 this 始终指向函数定义时的 this，而非执行时。

这是因为箭头函数中没有 this 绑定，必须通过查找作用域链来决定其值。如果箭头函数被非箭头函数包含，则 this 绑定的是最近一层非箭头函数的 this，否则 this 为 undefined。

例 8：

箭头函数为什么不可以作为构造函数被调用（即使用 new 命令）？因为：

  • 没有自己的 this，无法使用 call/apply。
  • 没有 prototype 属性 ，而 new 命令在执行时需要将构造函数的 prototype 赋值给新的对象的 __proto__。

在函数内部使用 _this = this

如果不使用 ES6，那么这种方式是最简单的方式。

我们是先将调用这个函数的对象保存在变量 _this 中，然后在函数中都使用这个 _this，这样 _this 就不会改变了。

我们将指向 a 的 this 赋值给一个变量 _this，这样在 func2() 中的 _this 就是指向对象 a 了。

使用 apply, call, bind

这三个函数也是可以改变 this 的指向的，原理稍后再讲，先来看一下如何实现：

apply

apply, call, bind 都可以改变 this 的指向，但是这三个函数稍有不同。

在 MDN 中 Function.prototype.apply() 的定义：

apply() 方法调用一个函数，其具有一个指定的 this 值，以及作为一个数组（或类似数组的对象）提供的参数。

  • thisArg：必选的。在 func 函数运行时指定的 this 值。需要注意的是，指定的 this 值并不一定是该函数执行时真正的 this 值，如果这个函数处于非严格模式下，则指定为 null/undefined 时会自动指向全局对象（浏览器中就是 window 对象），同时值为原始值（数字，字符串，布尔值）的 this 会被自动包装。
  • argsArray：可选的。一个数组或者类数组对象，其中的数组元素将作为单独的参数传给 func 函数。如果该参数的值为 null/undefined，则表示不需要传入任何参数。从 ES5 开始可以使用类数组对象。

被函数方法调用（call, apply）

在 JavaScript 中，函数是对象，有它的属性和方法。

Function.prototype.call() 和 Function.prototype.apply() 是预定义的函数方法，可用于调用函数，两个方法的第一个参数必须是对象本身。

例 6 的 innerFunction() 的调用属于第一种调用方式：作为函数被调用。它没有挂载在任何对象上，所以对于没有挂载在任何对象上的函数，在非严格模式下 this 就是指向 window 的。

例 7 可以简单理解为 匿名函数的 this 永远指向 window。

还是那句话 this 永远指向最后调用它的那个对象，那么就找最后调用匿名函数的对象，但是匿名函数没有名字，所以我们是没有办法被其他对象调用匿名函数的。所以匿名函数的 this 永远指向 window。

那匿名函数都是怎么调用的？

  1. 自执行，就是在匿名函数后面加 () 让其在定义完成之后直接执行。
  2. 作为其他函数的参数，比如例 7 中的 setTimeout()。
